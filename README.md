# 智能求职助手 (ZhiPin Helper)

Boss直聘自动化投递浏览器插件，支持智能筛选和AI辅助投递。

## 功能特性

- **薪资范围筛选** - 支持快捷预设和自定义输入，单位为K（千元）
- **公司黑名单** - 排除特定公司（如外包、不良企业）
- **工作内容匹配** - 关键字包含/排除筛选
- **公司规模筛选** - 选择合适的企业规模范围
- **学历要求筛选** - 过滤不符合学历要求的岗位
- **Boss活跃度过滤** - 过滤不活跃的Boss/职位
- **自定义AI模型** - 支持国内主流AI模型（通义千问、DeepSeek、Kimi等）
- **智能问候语** - 根据岗位JD自动生成个性化打招呼内容
- **自动回复消息** - AI自动回复HR的消息
- **反调试保护** - 绕过Boss直聘的开发者工具检测机制

## 技术栈

- **框架**: WXT (浏览器插件开发框架)
- **UI**: React + Tailwind CSS v4
- **语言**: TypeScript
- **存储**: Chrome Storage API

## 配置说明

### 筛选配置

在插件弹窗的"筛选"标签页中配置：
- **薪资范围**: 点击预设按钮快速选择，或手动输入数值（单位K）
  - 示例：输入 10 表示 10K = 1万元/月
  - 输入 15-20 表示期望薪资 15K-20K
- **Boss活跃时间**: 点击预设按钮或手动输入小时数
  - 不限：投递所有职位
  - 今日：只投递今日活跃的Boss
  - 3日内：Boss在72小时内活跃过
  - 本周：Boss在168小时内活跃过
- **公司规模**: 选择企业人数范围
- **学历要求**: 多选符合的学历要求
- **工作内容关键字**: 包含/排除的关键字筛选

### AI配置

在"AI配置"标签页中配置：
1. 选择AI提供商（通义千问、DeepSeek、Kimi等）
2. 输入API Key
3. 选择模型版本

### 简历设置

在"设置"标签页中：
- 粘贴简历内容（用于生成个性化问候语）
- 配置公司黑名单
- 启用/禁用自动投递

## 调试方法

Boss直聘有反调试机制，打开F12会导致页面闪烁退出。本插件已内置反调试保护，但仍建议使用以下调试方式：

### 方法1：Popup 诊断功能（推荐）

```
1. 打开 Boss直聘职位列表页
2. 点击插件图标打开 Popup
3. 点击「诊断」按钮
4. 查看诊断结果 JSON，包含：
   - jobLinksCount: 职位链接数量
   - salaryElements: 薪资元素列表
   - possibleJobCards: 职位卡片结构分析
   - jobSalaryDeepAnalysis: 薪资元素深度分析
```

诊断结果会自动保存到项目根目录的 `diagnosisResult.json` 文件。

### 方法2：使用扩展后台页面

```
1. chrome://extensions/
2. 找到「智能求职助手」
3. 点击「service worker: background.js」链接
4. 在打开的DevTools中查看日志输出
```

日志格式：
```
[智能求职助手] Background 已启动
[Content Script] Content Script 已加载（动态特征匹配模式）
检测到职位更新: 17 个职位
职位示例: 前端开发工程师 | 15-25K | XX公司
```

### 方法3：先打开DevTools再进入页面

```
1. 打开空白标签页
2. 按 F12 打开 DevTools
3. 保持 DevTools 打开
4. 在地址栏输入 Boss直聘网址
5. 反调试保护会阻止页面退出
```

                                    ↓
┌─────────────────────────────────────────────────────────────────────┐
│  4. Popup 展示符合条件的职位列表                                        │
│     用户可查看、勾选想要沟通的职位                                       │

                                    ↓
┌─────────────────────────────────────────────────────────────────────┐
│  5. 用户点击"自动打招呼"                                               │
│     系统调用 AI (通义千问/DeepSeek/Kimi等)                             │

│  6. 自动发送打招呼消息                                                 │
│     进入职位详情页 → 点击"立即沟通" → 输入文案 → 发送                    │

                                    ↓
┌─────────────────────────────────────────────────────────────────────┐
│  7. 监听 HR 回复消息                                                  │

                                    ↓

└─────────────────────────────────────────────────────────────────────┘

## 注意事项

- 本插件仅读取页面DOM数据，不调用后台API
- 使用**动态特征匹配**技术识别职位信息，不依赖固定class名
- 已内置反调试保护，可绕过Boss的开发者工具检测
- 建议设置合理的投递间隔（5-10秒），避免被检测
- AI功能需要用户提供自己的API Key
- 数据存储在本地，不上传服务器

## 数据抓取原理

本插件采用**DOM结构化选择器 + 动态特征匹配**方式识别职位信息。

### 已验证的 DOM 结构

通过诊断功能确认的 Boss直聘职位卡片 DOM 结构：

```
<li class="job-card-box">                  ← 职位卡片容器
  <div class="job-info">                   ← 职位信息区
    <div class="job-title clearfix">       ← 标题行
      <a class="job-name">前端开发工程师</a>  ← 职位名称（链接）
      <span class="job-salary">15-25K</span> ← 薪资
    </div>
    <div class="tag-list">                 ← 标签区
      <li>3-5年</li>                       ← 经验要求
      <li>本科</li>                        ← 学历要求
    </div>
  </div>
  <div class="job-card-footer">            ← 底部信息
    <a class="boss-info">XX公司</a>         ← 公司名称（链接）
    <span class="company-location">深圳·南山区</span> ← 地区
    <span class="boss-online-icon">今日活跃</span> ← Boss活跃状态
  </div>
</li>
```

### 数据提取策略

| 字段 | 选择器 | 备选方案 |
|------|--------|----------|
| **职位链接** | `a[href*="/job_detail/"]` | href 正则匹配 |
| **职位名称** | `.job-name` | 链接文本 |
| **薪资** | `.job-salary` | 文本正则 `\d+-\d+K` |
| **公司** | `.boss-info` | `/gongsi/` 链接 |
| **学历** | `.tag-list li` | 文本匹配 |
| **经验** | `.tag-list li` | 文本匹配 |
| **地区** | `.company-location` | 文本解析 |
| **Boss活跃** | `.boss-online-icon` | 文本匹配 |

### 注意事项

- **翻译插件干扰**: 如果页面有翻译插件（如 Relingo），可能导致薪资文本被替换。建议禁用翻译插件后再使用。
- **职位链接格式**: `/job_detail/{id}.html`，不是 `/job/`
- **动态加载**: 职位列表可能滚动加载，需监听 DOM 变化

## 备选方案：截图+LLM视觉识别（未实现）

当DOM抓取方案不可行时，可考虑以下备选方案：

### 方案原理

```
┌─────────────────────────────────────────────┐
│  工作流程                                    │
├─────────────────────────────────────────────┤
│  1. Content Script 截取页面截图              │
│  2. 发送截图到 Background                    │
│  3. Background 调用 LLM 视觉模型 API          │
│  4. LLM 返回职位卡片坐标和内容                │
│  5. Content Script 模拟点击对应位置          │
└─────────────────────────────────────────────┘
```

### Prompt 示例

```text
这是一张Boss直聘职位列表页面的截图。
请识别所有职位卡片的位置，返回JSON格式：
[
  {
    "index": 1,
    "x": 200,    // 卡片中心X坐标
    "y": 150,    // 卡片中心Y坐标  
    "title": "前端开发工程师",
    "salary": "15-25K",
    "company": "XX公司",
    "bossActive": "今日活跃"
  },
  ...
]
只返回JSON，不要其他说明。
```

### 方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **DOM动态匹配**（当前） | 无额外成本、速度快 | 可能因页面结构变化失效 |
| **截图+LLM识别**（备选） | 不依赖DOM、适应性强 | API成本高、延迟大 |

### 适用场景

截图+LLM方案适合以下情况：
- DOM结构大幅改版，无法通过特征匹配识别
- 需要理解复杂页面布局（如推荐页、详情页）
- 需要模拟更真实的人类操作行为

### 成本估算

| LLM视觉模型 | 单次识别成本 | 说明 |
|-------------|-------------|------|
| DeepSeek VL | ~0.01元 | 性价比高 |
| 通义千问 VL | ~0.02元 | 阿里云服务 |
| GPT-4V | ~0.05元 | 最贵但最准确 |

每页识别约消耗几分钱，高频使用成本较高。

### 支持的视觉模型

- DeepSeek Vision
- 通义千问 VL (qwen-vl-plus, qwen-vl-max)
- Claude Vision (需海外API)
- GPT-4V (需海外API)
